# 共有用要点（専務向け）: 車両リース管理自動化「承認/差戻し」追加で何が変わるか

- 作成日: 2026-02-02
- 参照: `docs/plan_vehicle_lease_renewal_approval_flow.md`

## 何が変わるか（要点）

- 一次確認（本部長/副本部長等）の回答が揃った後に、専務向けに「承認待ち」として結果をまとめて提示できるようにします
- 専務は「承認」または「差戻し（コメント付き）」を選べるようになります（差戻しが出た場合、誰へ戻して再回答してもらうかも運用に含めます）
- 承認結果は、車両管理担当へ「結果通知」として自動で届くようにします（以降のリース会社連絡は人手）
- 手続きが完了したものは「完了フラグ」として追跡できるようにします（抜け漏れ防止）

## 決定事項（打ち合わせ前提）

- 承認/差戻し: ハイブリッド（基本は依頼単位で承認、必要なら車両単位で差戻し）（Q2=C）
- 差戻し時の戻し先: 一次確認者へ戻す（本部長/副本部長へ再回答依頼）（Q3=A）
- 選択肢表現: 4択は維持しつつ「更新/解約」表現へ寄せる（Q4=C）

## 追加で確認したい（残タスク）

- 差戻しコメントの粒度（依頼全体で1つに固定するか／車両ごとコメントも必要か）

## 選択肢（表示イメージ）

- 更新（再リース）
- 解約（入替）
- 解約（満了）
- 未定
