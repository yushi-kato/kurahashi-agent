# 共有用要点（井上さん向け）: 車両リース管理自動化「承認/差戻し」追加で何が変わるか

- 作成日: 2026-02-02
- 参照: `docs/plan_vehicle_lease_renewal_approval_flow.md`

## 何が変わるか（要点）

- 「一次確認 → 専務承認（差戻しあり）→ 車両管理担当へ結果通知 → 完了フラグ」の段取りを、ツールのフローに追加します
- これまでの「管理部門に依頼して回答回収」だけではなく、一次確認者（本部長/副本部長等）と承認者（専務）を登場人物として扱います
- 差戻しが発生した時に「誰へ戻すか / どう再回答するか」を、データ（ステータス/コメント）として残せるようにします
- 最終的に「完了（リース会社へ連絡済み等）」を、車両単位または依頼単位で追えるようにします（完了フラグ追加）

## 決定事項（打ち合わせ前提）

- 一次確認の回答者: 本部長/副本部長（Q1=B）
- 専務承認の粒度: ハイブリッド（基本は依頼単位、必要なら車両単位で差戻し）（Q2=C）
- 差戻し時の戻し先: 一次確認者へ戻す（Q3=A）
- 選択肢ラベル: 4択は維持しつつ「更新/解約」表現へ寄せる（Q4=C）
- 送付タイミング: ローリング/バッチの両対応（Q5=C）
- 完了フラグ: 車両単位で完了（車両管理担当がチェック）（Q6=A）

## 追加で確認したい（残タスク）

- 一次確認者（本部長/副本部長）は「管理部門ごとに異なる」で良いか／「全社で固定」か
- 送付タイミング（バッチ運用）の具体（月/頻度。例: 2月に1年分、2月/8月の半期など）

## 選択肢（表示イメージ）

- 更新（再リース）
- 解約（入替）
- 解約（満了）
- 未定
